Context
One platform. Seven networks.
Qargo integrates directly with the UK's major pallet networks so depot operators can manage orders, statuses, PODs, ETAs, and cross-dock scanning from a single system instead of juggling multiple portals.
Two-way data flows — Qargo both sends and receives data with each network. Most features support import (network to Qargo) and export (Qargo to network) directions.

Network Deep Dive
Palletline — the deepest integration
Strengths
Full two-way everything — orders, statuses, ETAs, ePOD signatures via DSC FTP, PODs, COP import, and complete cross-dock scanning.

Fastest polling — order import every 2 mins, status polling every 1 min.

EPOD via FTP — signature, name, timestamp, and GPS coordinates exported via Palletline's DSC service.
Key Details
ETA rules: Import only when requesting depot. Export only after DPDV scan and when tenant is delivery depot.

COP: Import-only, available after ARRH hub scan.

Cross-dock codes: SCOT (coll), ARRD + SCDV (dlv inbound/outbound), ARRI, EPOD.
Network Deep Dive
Palletforce — full-featured via Alliance
Strengths
Updates + cancellations — one of few networks supporting order updates and cancellations after export.

Bidirectional COP — downloads blank COP, uploads signed COP back. Delivery images both ways.

Customer API — also supported for Palletforce alongside the depot integration.
Key Details
ETA export requires the ETA surcharge delivery extra on the stop. Sent as a 3-hour time window.

No POD document export — Palletforce strictly enforces ePOD or COP only via their API.

Cross-dock codes: SRID, SCOT, ARRD, DELV.
Network Deep Dive
Pallex / Fortec — Nexus-powered
Strengths
Updates + cancellations — order modifications supported via Nexus API.

Full document suite — ePOD, POD, COP, and delivery images all bidirectional.

Fast polling — orders every 5 mins, statuses every 1 min.
Key Details
ETA export via Optimize — exclusive Pallex agreement. ETAs sent to Optimize who pass them to Pallex. Limited to 2 per consignment.

Cross-dock: delivery scans only — code 14 (loaded into delivery vehicle).

Status codes: 39 (collected), 41 (at delivery), 37 (delivered).
Network Deep Dive
TPN — growing integration
Strengths
Full document exchange — ePOD, POD, COP, and delivery images all supported both ways.

Cross-dock scanning — collection depot scanned out, delivery depot scanned in/stored/scanned out.

Multi-depot supported.
Limitations
No order updates once exported.

No ETA export — TPN API doesn't support it.

Missing addresses — opposite leg address not exposed via API. Account code used as collection location alias.
Network Deep Dive
UPN & Palletways
UPN
Unique features: labels, pallet network routing via gazetteer, service mapping.
No COP or paper POD via API.
Status import limited — no import where tenant is delivering.
ETA export planned for 2026 with Variable ETAs feature.
Palletways
Hybrid API + FTP — order import via API (1hr) or FTP (1min).
Delivery-only export — no updates or cancellations.
No ETAs — API does not support them.
No COP — Palletways phasing it out.
Both networks support full cross-dock scanning for collection and delivery depot use cases, along with multi-depot operation.

Watch Out
Known limitations to keep in mind
Palletways ETAs
The Palletways API does not support ETAs in either direction. No workaround available.
TPN ETA Export
No API endpoint exists. TPN indicated it may come in 2026. Using Qargo driver app means losing TPN app ETAs.
UPN Status Import
Cannot import statuses where the tenant is delivering. This is a UPN API limitation.
UPN COP & Paper POD
UPN does not support COP or paper-based PODs via their API. ePOD only.
TPN Missing Addresses
Opposite leg address not exposed via API. Collection address missing for deliveries; delivery address missing for collections.
Palletforce POD Export
Palletforce strictly prevents POD document export via their API. Only ePOD or COP can be sent.
